# Checklist — Testing

## Objectif

Vérifier que les tests sont complets et conformes aux standards AEGIS avant validation.

## Quand utiliser

Avant toute Pull Request ou livraison de fonctionnalité.

## Prérequis

- Avoir lu `05-Engineering/Testing/RULES.md`

## Procédure

- [ ] Tests unitaires écrits et passants
- [ ] Tests d'intégration écrits et passants
- [ ] Tests métier (cas d'usage réels) définis
- [ ] Tests de sécurité définis
- [ ] Tests de performance définis
- [ ] Tests de non-régression en place
- [ ] Couverture de code acceptable
- [ ] Les cas limites sont couverts
- [ ] Les erreurs attendues sont testées
- [ ] Aucun test ignoré sans justification

## Validation

Tous les points doivent être cochés avant de considérer le livrable comme terminé.
