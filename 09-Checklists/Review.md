# Checklist — Review

## Objectif

Guider la revue d'un livrable (code, document ou décision) avant validation finale.

## Quand utiliser

Pendant toute revue effectuée par un agent IA ou un humain.

## Prérequis

- Avoir lu `01-Kernel/REVIEW_PROTOCOL.md`
- Avoir lu `03-Orchestration/REVIEW_PIPELINE.md`

## Procédure

- [ ] Le besoin initial est compris et respecté
- [ ] L'architecture est respectée
- [ ] Le code est simple et lisible
- [ ] Aucune duplication
- [ ] Les erreurs sont correctement gérées
- [ ] Les performances sont acceptables
- [ ] La documentation est complète et mise à jour
- [ ] Les risques sont identifiés
- [ ] Les checklists de domaine correspondantes ont été validées
- [ ] Aucun TODO oublié
- [ ] Aucun fichier mort
- [ ] Aucune référence cassée
- [ ] La migration Supabase est respectée si applicable

## Validation

Tous les points doivent être cochés avant d'approuver le livrable.
