# 06-Playbooks

## Objectif

Décrire comment réaliser UNE tâche récurrente, étape par étape.
Un Playbook = une tâche. Un Playbook = un résultat prévisible.

## Structure

```
06-Playbooks/
├── README.md
├── Architecture/
├── Backend/
├── Frontend/
├── Database/      ← Playbooks Supabase
├── OCR/
├── AI/
├── Security/
├── Testing/
└── Documentation/
```

## Format obligatoire de chaque Playbook

```
# Objectif
# Préconditions
# Étapes
# Résultat attendu
# Validation finale
```

## Comment utiliser

Identifier la tâche à réaliser, trouver le Playbook correspondant, suivre les étapes.
