# Template — Issue

## Objectif

Uniformiser la création des issues pour faciliter la planification et l'exécution.

## Quand utiliser

À chaque signalement de bug, demande de fonctionnalité ou tâche technique.

## Prérequis

Aucun.

## Procédure

---

```markdown
## Titre

[Titre clair et concis]

## Type

- [ ] Bug
- [ ] Fonctionnalité
- [ ] Documentation
- [ ] Tâche technique

## Description

[Décrire clairement le problème ou la demande.]

## Comportement attendu

[Décrire ce qui devrait se passer.]

## Comportement actuel

[Décrire ce qui se passe actuellement (pour les bugs).]

## Contexte

- Module concerné : [Nom du module]
- Environnement : [Dev | Staging | Production]

## Critères d'acceptation

- [ ] [Critère 1]
- [ ] [Critère 2]

## Dépendances

- [Issue #X]
```

---

## Validation

L'issue est prête lorsque les critères d'acceptation sont définis.
