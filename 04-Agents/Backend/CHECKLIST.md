# Agent Backend — Checklist

## Objectif

Vérifier qu'un livrable Backend est complet avant soumission.

## Checklist

- [ ] Feature Contract respecté
- [ ] Architecture respectée
- [ ] Supabase utilisé exclusivement
- [ ] Aucune référence à PostgreSQL local, SQLite, Firebase, MongoDB ou Prisma
- [ ] Toutes les erreurs gérées
- [ ] Logs en place
- [ ] Validation des entrées effectuée
- [ ] Tests écrits et passants
- [ ] Documentation endpoint mise à jour
- [ ] Aucun TODO oublié
- [ ] PR conforme à `10-Templates/PR.md`
