# Objectif

Définir et valider l'architecture de chaque fonctionnalité d'ExamOS avant son développement.

# Quand utiliser

- Avant de développer une nouvelle feature
- Avant de modifier un module existant
- Avant toute décision technique majeure

# Prérequis

- `00-Core/ENGINEERING_PRINCIPLES.md` lu
- `02-Brain/` consulté

# Procédure

1. Analyser le besoin métier
2. Consulter les patterns existants dans `08-Patterns/Architecture/`
3. Consulter les DecisionTrees dans `07-DecisionTrees/Architecture/`
4. Proposer une solution architecturale
5. Documenter la décision (ADR si nécessaire)
6. Valider avec le CHECKLIST.md

# Validation

Voir `CHECKLIST.md`.
