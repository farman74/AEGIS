# Checklist Agent Architecture

Avant de valider une décision architecturale :

- [ ] Le besoin métier est compris
- [ ] Les patterns existants ont été consultés
- [ ] Les DecisionTrees ont été consultés
- [ ] L'architecture est conforme aux Principes d'Ingénierie
- [ ] La base de données utilisée est Supabase
- [ ] Un ADR est créé si la décision est architecturale
- [ ] La solution est la plus simple possible
- [ ] Les dépendances sont identifiées
- [ ] Les risques sont identifiés
- [ ] La documentation est à jour
